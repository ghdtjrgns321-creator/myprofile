Core Tech & Tools

FastAPI PostgreSQL AWS EC2 (m7i-flex) SSH Tunneling GitHub Actions Docker Flutter

Key Engineering Achievements (STAR)

1. AWS 인스턴스 최적화 및 Hybrid Cloud 구축 전략

Situation: 초기 Free Tier(`t3.micro`) 환경에서 FastAPI 서버와 Dockerized PostgreSQL을 동시 구동하자 메모리 부족(OOM) 및 레이턴시 저하가 발생했습니다. 또한, 예산 제약으로 고성능 클라우드 GPU(g4dn 등) 사용이 불가능했습니다.

Task: 최소한의 비용으로 Web/DB 서버의 안정성을 확보하고, 동시에 고성능 AI 추론을 수행할 수 있는 아키텍처를 설계해야 했습니다.

Action:

  • Vertical Scaling (Scale-Up): 벤치마크 테스트 후, Web/DB 부하를 감당하면서도 가성비가 우수한 `m7i-flex.large` 인스턴스로 마이그레이션하여 서버 안정성을 확보했습니다.
  • Hybrid Cloud Tunneling: 고비용 GPU 인스턴스 대신, 로컬 워크스테이션(RTX 3060)을 SSH Reverse Tunneling으로 AWS와 연결하여 보안 그룹 개방 없이 0원으로 추론 자원을 확보했습니다.

Result: 서버 크래시 문제를 100% 해결하고, 클라우드 비용을 최소 유지비 수준으로 억제하면서도 AI 추론 성능을 온전히 활용하는 효율적인 파이프라인을 완성했습니다.

2. GitHub Actions 기반 CI/CD 및 무중단 배포 환경 구축

Situation: 잦은 코드 업데이트 과정에서 수동 배포(`git pull` -> `kill` -> `nohup`) 방식은 Human Error 위험이 높았고, 배포 시마다 서비스 다운타임이 발생했습니다.

Task: 개발 생산성을 위해 코드 푸시 시 자동으로 테스트 및 배포가 이루어지는 파이프라인 구축이 필요했습니다.

Action:

  • Automated Pipeline: GitHub Actions를 도입하여 `main` 브랜치 병합 시 AWS EC2에 자동 배포되도록 워크플로우를 작성했습니다.
  • Systemd Integration: FastAPI 애플리케이션을 리눅스 서비스(`systemd`)로 등록하여, 배포 후 자동 재시작 및 프로세스 비정상 종료 시 자가 복구(Auto-healing)가 가능하도록 설정했습니다.

Result: 배포 시간을 단축하고, 개발자가 인프라 관리보다 기능 구현에 집중할 수 있는 환경을 조성했습니다.

3. 비동기 API 서버 구축 및 협업 효율화

Situation: 프론트엔드 팀과의 협업 과정에서 명확한 API 명세가 필요했으며, AI 추론 지연 시간 동안 웹 서버가 블로킹(Blocking)되지 않아야 했습니다.

Task: 고성능 비동기 처리가 가능한 서버를 구축하고, 인증 로직이 포함된 테스트 환경을 제공해야 했습니다.

Action:

  • Async Architecture: `FastAPI`와 `Async SQLAlchemy`를 채택하여 I/O 블로킹을 최소화하고 동시 접속 처리량을 극대화했습니다.
  • Swagger UI 개선: 기존의 번거로운 Form 인증 대신 `HTTPBearer` 방식을 적용하여, Access Token 입력만으로 즉시 API 테스트가 가능한 환경을 프론트엔드 팀에 제공했습니다.
  • RAG System Integration: 단순 진단 결과(JSON)를 넘어, LangChain을 통해 의학 가이드라인 기반의 맞춤형 조언을 생성하는 로직을 백엔드에 통합했습니다.

Result: 프론트엔드-백엔드 간 연동 시행착오를 대폭 줄이고, 확장성 있는 데이터 처리 파이프라인을 구축했습니다.

Project Outcome

  • Full Pipeline 구축: Flutter(앱) - AWS(서버) - Local(AI) - DB로 이어지는 End-to-End 서비스 완성
  • 비용 효율성 극대화: 클라우드 비용을 최소 유지비(EC2 t/m시리즈) 수준으로 절감하며 GPU 자원 활용
  • 사용자 중심 기능: 객관적 수치(KGS 등급) 제공 및 즉각적인 의료 가이드라인 제시로 1차 스크리닝 도구로서의 가치 입증
