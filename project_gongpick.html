GongPick: 공무원 업무추진비 기반 맛집 추천

비정형 데이터 분석 및 Azure VM 기반 인터랙티브 웹 서비스 구축

GitHub Repo 발표 영상 보기 (Presenter) 발표 자료 (PDF)

My Role

  • Data Engineering & Modeling

    공공데이터(업무추진비)의 비정형 텍스트(가게명, 지출내역) 정제 및 결측치 처리. Random Forest 기반의 상황별(인원, 예산, 계절 등) 맛집 추천 모델링 수행.

  • Cloud Infrastructure (Azure)

    Azure VM(Ubuntu) 인스턴스 프로비저닝 및 네트워크 보안 그룹(NSG) 설정을 통한 외부 접속 환경 구성. 리눅스 환경에서의 패키지 관리 및 배포 담당.

  • Rapid Application Development

    Streamlit을 활용하여 별도의 프론트엔드/백엔드 구축 없이 데이터 분석 결과를 즉시 시각화하고, 챗봇 형태의 인터페이스를 빠르게 구현하여 MVP(Minimum Viable Product) 완성.

Technologies Used

Python Azure VM Streamlit Scikit-learn Pandas

Development Process

1. 데이터 수집 및 전처리 (Data Pipeline)

전국 구청의 업무추진비 집행 내역을 수집하여, '드림스타트 사례관리 관계자 간담회,연회다원,3명,16500'와 같은 비정형 텍스트 데이터에서 핵심 키워드(식당명, 인원, 금액)를 추출하고 정규화(Normalization)하는 전처리 파이프라인을 구축했습니다.

2. 모델링 및 로직 구현

단순 랭킹이 아닌 '상황(Context) 기반 추천'을 위해 계절, 시간대, 인원수, 예산 데이터를 Feature로 활용하여 분류 모델을 학습시켰습니다.

3. 서비스 배포 (Azure VM + Streamlit)

복잡한 웹 프레임워크 대신 Streamlit을 채택하여 데이터 로직 구현에 집중했습니다. Azure VM(Standard D8as v5)을 할당받고 방화벽(Inbound Port 8501)을 개방하여, 로컬이 아닌 클라우드 환경에서 누구나 접속 가능한 챗봇 서비스를 띄웠습니다.

Lesson Learned

"'기술의 선택은 목적에 부합해야 한다'는 것을 배웠습니다. 제한된 프로젝트 기간 내에 데이터 분석 모델의 가치를 보여주기 위해, 프론트/백엔드를 분리하는 오버엔지니어링 대신 Python만으로 빠르게 앱을 빌드할 수 있는 Streamlit을 선택하여 개발 생산성을 극대화했습니다."

"또한 로컬 환경(localhost)과 클라우드 환경(Azure VM)의 차이를 몸소 겪으며, 리눅스 환경 설정, 권한 관리, 네트워크 포트 포워딩 등 서비스 배포를 위한 인프라 기초 역량을 다질 수 있었습니다."
