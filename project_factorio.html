GCP 기반 Linux 게임 서버 구축

온프레미스에서 클라우드(IaaS)로의 서비스 마이그레이션 및 24/7 운영

GCP Compute Engine Debian Linux VPC/Firewall Bash Shell

Context & Problem

지인들과 'Factorio'라는 게임을 즐기던 중, P2P 연결 방식의 한계를 겪었습니다. 호스트(Host) 플레이어가 접속을 종료하면 다른 사람들도 게임을 할 수 없었고, 호스트 컴퓨터의 네트워크 상태에 따라 랙(Lag)이 발생했습니다.

핵심 문제: 단일 장애 지점(SPOF)인 호스트 의존성 및 가용성 부족

Implementation Details

1. 클라우드 리소스 프로비저닝

GCP에서 e2-medium 인스턴스를 생성하고 Debian Linux OS를 설치했습니다. 게임 서버의 메모리 요구사항을 분석하여 스왑 메모리(Swap Memory)를 설정해 비용 효율적인 스펙을 구성했습니다.

2. 네트워크 및 보안 설정

외부 접속을 위해 고정 IP(Static IP)를 할당받고, VPC 방화벽 규칙에서 게임이 사용하는 UDP 34197 포트를 개방했습니다. SSH 접속 보안을 위해 키 기반 인증 방식을 적용했습니다.

3. 프로세스 관리 및 자동화

SSH 세션이 끊겨도 서버가 유지되도록 nohup을 사용해 백그라운드 데몬으로 실행했습니다. 또한 run.sh, stop.sh 스크립트를 작성하여 서버 제어를 간편화했습니다.

Lesson Learned

이 프로젝트는 단순한 게임 서버 구축을 넘어, "서비스를 내 컴퓨터 밖(Cloud)으로 옮길 때 고려해야 할 인프라 요소들"을 몸으로 익힌 경험이었습니다. 리눅스 파일 시스템 권한, 포트 포워딩, 백그라운드 프로세스 관리 등 엔지니어링의 기초 체력을 실제 운영(500시간 이상 무중단)을 통해 다질 수 있었습니다.
