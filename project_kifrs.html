환각 방지 5-Layer Pipeline

전 과정(데이터 적재 -> 최종 답변)을 통제하는 확정적(Deterministic) 아키텍처

5-Layer Data Pipeline

  1. 1 도메인 데이터 적재
    1115호 본문 및 관련 사례와 지적사례 1,575건을 DB에 저장
    (본문·적용사례·질의회신·감리사례·KAI교육·토픽큐레이션)
  2. 2
    핀포인트 + Reranker 검색
    • • Tier 1 핀포인트 —사전 배정문서 ID로 DB 직접 조회. 검색 엔진 경유X → 핵심 근거 누락 0%
    • • Tier 2 하이브리드 보충 — 약칭 자동 인식 + 가중치 적용(본문 1.3 / 감리지적 1.2 / BC 0.8)
    • • Cohere Cross-encoder 재평가 — LLM을 통한 관련성 재평가
  3. 3
    듀얼 LLM 라우팅
    • • Gemini Flash — 회계 추론 1위 (81%). 일반·상황 질문 전담
    • • gpt-4.1-mini — 산술 정확도 1위 (100%). 계산 질문 전담
  4. 4 Decision Tree 강제
    의사결정 트리로 AI 자의적 추론 차단
    정보 부족 시 임의 판단 대신 조건부 결론(Case 분기) 도출
  5. 5 PydanticAI 구조화 출력
    '근거'와 '결론' 분리를 코드 수준에서 강제

동일 질문

"A사가 B에게 재화를 100원에 공급하고 세금계산서를 발행, B가 고객 C에게 120원에 판매할 때 A의 매출액은?"

6개 LLM 모두 "100원"으로 단정
ㅡ 본인/대리인 여부를 확인하지 않고 결론을 내림 및 근거 문단 제시 미흡함.
ㅡ 이 시스템은 체크리스트로 조건을 확인한 뒤 Case 1(본인: 120원) / Case 2(대리인: 100원)로 조건부 분기하고, 기준서 문단을 인용.

이 시스템의 답변

  • "100원인지 120원인지"를 단정하지 않고 확인 질문을 먼저 제시
  • Case 1(본인: 120원) / Case 2(대리인: 100원) 조건부 분기로 결론 도출
  • 기준서 문단 B35, B37 등 구체적 근거 인용 + 감리사례 경고 포함

PDF 전처리는 최후의 수단. 이미 구조화된 데이터(API, HTML)가 존재하는지 먼저 조사해야 한다.

문제

동일 질문에 매번 다른 형식의 답변. 프롬프트를 엄격하게 쓸수록 오히려 오류 증가.
LangGraph의 자유형 텍스트 출력이 비결정성의 근본 원인.

해결

LangGraph/LangChain → PydanticAI 전환. 프롬프트 수준의 부탁이 아닌 코드 수준의 강제.

지표 LangGraph PydanticAI
포맷 준수율불안정100%
인용 누락간헐적자동 재시도 → 0건
테스트 재현성3회 모두 상이형식 100% 동일

회고

정확성이 중요한 도메인에서는 코드수준에서의 강제가 필요하다.

문제

PydanticAI로 형식은 통제했으나 내용이 매번 변동됨.
같은 질문에 인용 문단, 결론 등을 AI가 매번 다르게 자의적으로 생성.

해결

26개 토픽 각각에 7개 섹션의 Decision Tree를 사전 정의.
AI는 정해진 체크리스트를 확인하고, 정해진 결론 분기 중 선택하며, 사전 배정된 문서만 인용 가능.

지표 PydanticAI만 + Decision Tree
검색 적중률36%76% (+40%p)
꼬리질문 일관성60%100%
환각 발생간헐적0건 (301회 검증)

회고

루트 사전 주입 및 코드 강제를 통해 변동성 차단 가능

문제

브랜치 전환 시 unstaged 변경사항 전체 소실. 불필요 파일 커밋
동일 문제 반복 디버깅, 에이전트에게 매번 같은 설명 반복.

해결

git 관리 규칙, 안전수칙 체계화.
CLAUDE.md·debugging.md·DECISION.md 등 문서 체계를 구축

회고

문서화를 통해 안전장치 확보 및 효율적인 작업이 가능하다는것을 깨달음.
